The main cause of this time increase/bottleneck is the size of MAX_MESSAGE. As the program can only transfer
that amount of bytes per turn, the number of turns exponentially increases with file size.
If MAX_MESSAGE is increased, then the number of turns would decrease thereby decreasing execution time.
